# English

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, send an email to **whchoi98@gmail.com** with:

1. Description of the vulnerability
2. Steps to reproduce
3. Potential impact
4. Suggested fix (if any)

You will receive an acknowledgment within 48 hours. We will work with you to understand and address the issue before any public disclosure.

## Scope

This policy applies to the project-init plugin and its generated project structures, including:

- Command files (`commands/*.md`)
- Generated hook scripts (`check-doc-sync.sh`, `commit-msg`)
- Generated `settings.json` configurations
- Reference templates

## Supported Versions

| Version | Supported |
|---------|-----------|
| 2.x     | Yes       |
| < 2.0   | No        |

# 한국어

## 취약점 보고

이 프로젝트에서 보안 취약점을 발견하셨다면 책임감 있게 보고해 주세요.

**보안 취약점에 대해 공개 GitHub 이슈를 생성하지 마세요.**

대신 **whchoi98@gmail.com**으로 아래 내용을 포함하여 이메일을 보내 주세요:

1. 취약점 설명
2. 재현 방법
3. 잠재적 영향
4. 권장 수정 방법 (있는 경우)

48시간 이내에 확인 응답을 드립니다. 공개 전에 문제를 함께 이해하고 해결하겠습니다.

## 범위

이 정책은 project-init 플러그인과 생성되는 프로젝트 구조에 적용됩니다:

- 커맨드 파일 (`commands/*.md`)
- 생성되는 훅 스크립트 (`check-doc-sync.sh`, `commit-msg`)
- 생성되는 `settings.json` 설정
- 참조 템플릿
